What is corewar ?

CoreWar is a programming game in which two or more computer programs compete for control of a virtual machine called MARS, which stands for Memory Array Redcode Simulator.

These programs are written in an assembly language called Redcode.

The goal of the game is to terminate all instances of the opposing program(s) in order to be the last one to execute.

How to play ?

On the current website, you can choose your champions in the form of ".s" files.

You can modify your champions using the "edit" mode, which allows you to make changes. I encourage you to explore it and discover its features on your own.

Once you have chosen your champions, you simply need to launch the game.

A window of your terminal will be displayed, and the game is built using CSFML.

Its history

The idea for Core War originated from Creeper and Reaper, the first computer worms created in the early 1970s by Bob Thomas and Ray Tomlinson.

The game originally formed part of a larger game called Darwin, developed by Victor A. Vyssotsky, Robert Morris Sr., and M. Douglas McIlroy at Bell Labs in the 1960s.

The term "core" derives from the technology of magnetic core memory, which was used at the time. It refers to the small magnetic rings, or cores, used to store data in early computers. The term "core dump" in computer jargon also has the same origin, referring to the process of dumping the contents of memory for debugging or analysis purposes.
